What happens when you initialize a repository? Why do you need to do it?

	To initialize a repository you must run “git init” to show any commits and running  “git status” will show you all of the files in the folder. This will show you all files including .git files that aren’t seen unless ran through the prompt.

How is the staging area different from the working directory and the repository?
What value do you think it offers?

	The staging area allows you to add commits as where the working directory does not, but both allow you to use git status to view the files from the staging area to the working directory. This offers you the ability to see the changes you’ve made.
